# 📊 微信朋友圈分析工具

## 📌 项目简介

本项目旨在将“发朋友圈”这一日常社交行为进行量化建模，构建一个**收益-风险评估系统**，帮助用户在发布内容前进行理性决策。

通过对受众结构、关系复杂度以及内容特征的分析，系统可以估算一条朋友圈可能带来的**正向收益与潜在风险**，并给出是否建议发布的判断。

---

## 🧠 核心思想

将朋友圈发布行为抽象为一个效用函数（Utility Function）：

U = Σ p_i (B_i - C_i · k_i)

其中：

- p_i：第 i 类受众占比  
- B_i：该类受众带来的收益（认同、互动、形象提升等）  
- C_i：潜在成本（误解、嫉妒、隐私暴露等）  
- k_i：关系复杂度（关系越复杂，风险放大系数越高）

最终 U 表示发布该朋友圈的**净收益评分**。

## ⚙️ 功能设计（MVP）

### 1️⃣ 朋友圈内容输入

用户可输入或选择：

- 内容类型（如：日常记录 / 成就展示 / 情绪表达 / 旅游 / 健身等）
- 发布时间（早晨 / 白天 / 晚间 / 深夜）
- 是否包含敏感信息（如：金钱、情绪、关系、定位、政治观点、工作机密、健康信息等）
- 朋友圈文案（可选）：系统会自动分析文案并参与风险修正

---

### 2️⃣ 受众结构建模

用户自定义受众分类及占比，例如：

- 家人
- 亲密朋友
- 普通朋友 / 同学
- 不熟联系人
- 特殊对象（如前任、暧昧对象等）

---

### 3️⃣ 收益与风险评估

对每类受众，系统会先按“受众类型”自动估算一组基线参数（白话版）：

- 好处分（原收益 B_i）：
  - 这类人看到后会不会更认可你  
  - 会不会让关系更近  
  - 会不会提升你的形象  

- 风险分（原成本 C_i）：
  - 会不会被误解  
  - 会不会引发比较 / 嫉妒  
  - 会不会造成信息泄露  

- 关系敏感度（原关系复杂度 k_i）：
  - 低：关系稳定，容易理解你  
  - 中：普通社交关系  
  - 高：关系复杂，容易放大风险

---

### 4️⃣ 评分与结果输出

系统输出：

- 📊 综合得分（百分制 0-100）  
- ⚠️ 风险贡献分析（按人群拆分）  
- 💡 发布建议：
  - ✅ 建议发布
  - ⚠️ 谨慎发布
  - ❌ 不建议发布

---

### 5️⃣ 分组可见模拟（已实现）

系统会自动比较三种可见方案并给出最优建议：

- 全部可见
- 仅分组可见（由“分组可见”勾选决定）
- 屏蔽指定人群（通过“屏蔽哪些人群”输入）

输出每个方案的评分与建议，并标记当前最优方案。

---

### 6️⃣ 可视化（可选扩展）

- 雷达图（收益 vs 风险）
- 柱状图（不同人群影响）
- 饼图（受众结构）

## ⚠️ 项目定位说明

本项目为**决策辅助工具**，并非真实预测系统。

其输出结果基于用户输入参数和建模假设，仅用于：

- 提供思考框架  
- 辅助判断风险  
- 提升表达策略  

不保证对现实社交结果的准确预测。

---

## 📖 项目意义

在社交媒体环境中，信息传播具有不确定性与复杂性。本项目尝试通过建模方式：

- 理解“表达”与“风险”的关系  
- 探索人际互动中的隐性成本  
- 提供一种理性化的社交决策思路

## ✅ 当前已实现（MVP v1.2.0）

- 提供 Web 表单输入：
  - 内容类型
  - 发布时间
  - 敏感信息标签
  - 受众人群参数（大概占比、受众类型、关系敏感度、是否在可见分组）
  - 屏蔽人群输入（用于分组可见模拟）
- 后端实现效用函数计算：
  - `U = Σ p_i (B_i - C_i · k_i)`（其中 `B_i / C_i` 由算法按受众类型自动估算，并叠加内容类型、发布时间、敏感信息修正）
- 输出结果包括：
  - 综合得分（百分制）`Utility Score`
  - 原始效用值 `U`（用于解释模型计算）
  - 三维风险画像（误解风险 / 关系风险 / 隐私风险）
  - 自动行动建议（发布时间、敏感信息降维、可见分组优化）+ 预计提分幅度
  - 一键应用建议并重算（在结果页直接应用建议动作）
  - 参数锁定与方案对比（A/B 方案保存、应用、并排对比）
  - 分数解释增强（拉低分数 Top 因子 + 贡献比例 + 影响方向）
  - 风险贡献 Top 3 人群
  - 人群明细贡献拆分
  - 发布建议（建议发布 / 谨慎发布 / 不建议发布）
  - 分组可见方案对比（全部可见 / 仅分组可见 / 屏蔽指定人群）
  - 历史记录与趋势统计（最近 N 次得分、风险等级分布、高风险标签统计）
  - 历史记录一键清空（带二次确认）
  - 个性化参数校准（偏保守 / 偏乐观 / 准确反馈）

## [1.2.0] - 2026-03-28

### 新增
- 新增历史记录落库模块 `moments/history.py`（SQLite），评估后自动写入分析结果。
- 新增 `GET /api/history?limit=N` 接口，支持最近 N 次趋势与高风险标签统计。
- 新增历史记录清空能力：`POST /api/history/clear`，结果页支持一键清空历史记录（含二次确认）。
- 结果页新增“历史记录与趋势分析”模块，展示最近 N 次得分趋势、风险等级分布和高风险标签聚集情况。
- 新增个性化校准模块 `moments/personalization.py`，支持基于用户反馈动态更新个人收益/风险权重。
- 新增 `GET /api/personalization-profile` 和 `POST /api/feedback` 接口。
- 结果页新增个性化反馈入口（偏保守 / 偏乐观 / 准确），反馈后可立即重算生效。
- 开始分析页新增填写进度条（实时百分比与下一步提示）。
- 新增草稿自动保存与自动恢复能力（本地草稿）。
- 新增未提交参数变更离页提醒，降低误操作丢失风险。
- 开始分析页重构为分步向导（内容设置 -> 受众设置 -> 确认并分析），避免一次展示全部填写项。
- 分步向导进一步细化为 6 步（内容类型、发布时间、敏感信息、文案、受众、可见与确认），进度条与步骤推进强绑定。
- 开始分析页流程进一步减负：删除重复进度展示，保留单一进度条并简化顶部引导文案。

### 变更
- `evaluate_request` 接入历史写入流程，评估结果 `meta` 增加历史记录写入信息。
- `README.md` Roadmap 状态更新：第 4 项“历史记录与趋势分析”标记为已完成。
- 评分引擎接入个性化收益/风险倍率，历史反馈会影响后续评估与建议模拟。

### 文档
- 更新 `README.md` 技术栈为当前实际使用（Flask + Vanilla JS + SQLite）。
- 更新 `README.md` API 列表与示例请求，补充历史记录和个性化相关接口。

## [1.1.0] - 2026-03-28

### 新增
- 新增文案智能分析能力：支持输入朋友圈文案后自动输出情绪强度、炫耀程度、隐私强度。
- 新增文案分析结果展示卡片：展示手动标签、自动识别标签与最终生效标签。
- 新增后端文案分析模块 `moments/copywriting.py`，基于中文关键词进行标签识别与风险维度增益计算。

### 变更
- 评估主链路支持 `copyText` 入参，并将文案分析结果参与评分、解释、可见方案模拟和行动建议。
- 扩展文案识别词库为中文主导，覆盖更常见朋友圈表达（情绪、炫耀、隐私、关系、工作、家庭等）。
- 更新路由/页面资源组织：采用 `static/js/*` 模块化脚本加载。

### 文档
- 更新 `README.md`，同步“文案输入”能力与中文词库驱动的参数修正说明。
- 将 Roadmap 第 5 项“文案智能分析”状态更新为已完成。

## [1.0.0] - 2026-03-27

### 新增
- 新增三页面流程：
  - `/` 首页
  - `/start-analysis` 参数填写页
  - `/analysis-result` 分析结果页
- 新增分组可见方案对比：
  - `all_visible`
  - `group_only`
  - `hide_selected`
- 新增更多敏感信息选项及对应后端风险系数。
- 新增结果页人群明细分页功能。
- 新增结果页行动建议模块。

### 变更
- 将受众输入模型从“手动填写好处分/风险分”改为“按受众类型自动估算”。
- 将占比输入从数字框改为滑动条（百分比）。
- 将评分输出改为百分制（0-100），并保留原始效用值。
- 将风险模型从单维风险升级为三维风险画像：
  - 误解风险
  - 关系风险
  - 隐私风险
- 将建议引擎升级为支持建议提分预估（`estimatedDeltaScore`）。
- 首页调整为内容居中的落地页布局，优化信息层级。
